# 아이폰 설정화면 버튼 안 눌리는 버그 수정

## 작업일: 2026-02-02

## 원인

### 터치 스크롤 감지 임계값이 너무 낮음 (5px)
- 설정 화면은 `scrollMaxY = 260`으로 스크롤 가능
- `handleTouchMove`에서 손가락 이동이 5px 초과하면 `_isTouchScrolling = true`
- 아이폰에서 버튼을 탭할 때 손가락 미세 이동이 5px 이상인 경우가 대부분
- `touchend`에서 `_isTouchScrolling === true`면 `handleInput`을 건너뜀
- 결과: 설정 화면의 모든 버튼(API 키, AI 생성, 사운드, 언어 등)이 거의 안 눌림

### 비스크롤 화면에서는 문제없던 이유
- 메인/전투/던전선택 화면은 `scrollMaxY = 0`
- `handleTouchStart`가 즉시 return → `_isTouchScrolling`이 절대 true가 안 됨

## 수정 내용

### `app/src/game/Game.js` - `handleTouchMove()`
1. 임계값 5px → **15px** 상향
   - 모바일에서 일반 탭 시 미세 이동은 보통 5-12px
   - 의도적 스크롤은 보통 20px 이상
   - 15px은 탭과 스크롤을 안정적으로 구분하는 값

2. 스크롤 판정 전에는 `scrollY`를 변경하지 않도록 수정
   - 기존: 1px 이동해도 즉시 `scrollY` 변경 (화면 떨림)
   - 수정: `_isTouchScrolling === true`일 때만 `scrollY` 변경
   - 탭 시 화면이 미세하게 흔들리는 문제도 해결

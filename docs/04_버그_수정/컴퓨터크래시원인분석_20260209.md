## 원인 해석

**VIDEO_TDR_FAILURE**는 GPU가 허용된 시간(기본 2초) 내에 작업을 완료하지 못했을 때 Windows가 GPU 드라이버를 리셋하려다 실패하면서 블루스크린이 발생하는 것입니다.

주요 가능 원인:
1. **NVIDIA 그래픽 드라이버 불안정** - 현재 드라이버(2025-11-04 버전)에 버그가 있을 수 있음
2. **GPU 과열** - 노트북 환경에서 쿨링이 불충분할 경우 GPU가 응답 불가 상태에 빠질 수 있음
3. **GPU 과부하** - 새벽 2시에 GPU를 많이 사용하는 작업(게임, 렌더링 등)이 실행 중이었을 가능성
4. **전원 공급 문제** - 노트북 배터리/어댑터 문제로 GPU에 충분한 전력이 공급되지 않았을 가능성

## 권장 조치

### 즉시 조치
1. **NVIDIA 드라이버 업데이트** - GeForce Experience 또는 NVIDIA 공식 사이트에서 최신 드라이버로 업데이트
2. **GPU 온도 모니터링** - HWMonitor, GPU-Z 등으로 GPU 온도 확인 (85도 이상이면 과열)

### 추가 조치
3. **노트북 쿨링 환경 개선** - 통풍구 청소, 쿨링 패드 사용 검토
4. **전원 어댑터 확인** - 충전기 연결 상태에서 작업 권장
5. **TDR 타임아웃 조정** (임시 방편) - 레지스트리에서 `TdrDelay` 값을 늘릴 수 있으나, 근본 해결은 아님

### 재발 시
- 크래시 발생 직전 실행 중이던 프로그램 기록
- GPU 온도 모니터링 결과 함께 확인
- NVIDIA 드라이버 클린 설치(DDU 사용) 고려

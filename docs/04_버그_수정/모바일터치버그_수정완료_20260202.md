# 모바일 터치 버튼 안 눌리는 버그 수정

## 작업일: 2026-02-02

## 원인 분석

### 핵심 버그: `scrollMaxY` 미리셋
- `SettingsScreen.js`에서 `game.scrollMaxY`를 양수로 설정
- `changeScreen()`에서 `scrollY = 0`은 리셋하지만 `scrollMaxY`는 리셋하지 않음
- 설정 화면 방문 후 다른 화면으로 이동해도 `scrollMaxY > 0` 유지
- 결과: `handleTouchStart()`가 스크롤 모드로 진입하고, 모바일에서 흔한 미세 손가락 이동(>5px)이 `_isTouchScrolling = true`를 발생시켜 모든 탭이 무시됨

### 보조 문제: 캔버스 `touch-action` 미지정
- 브라우저가 터치 이벤트를 기본 제스처(스크롤, 줌)로 소비할 가능성

## 수정 내용

### 1. `app/src/game/Game.js`
- `changeScreen()`에서 `this.scrollMaxY = 0` 추가
- 각 화면의 render 함수가 필요 시 다시 설정하므로 안전

### 2. `app/src/style.css`
- `#gameCanvas`에 `touch-action: none` 추가 (브라우저 기본 터치 제스처 차단)
- `-webkit-touch-callout: none` 추가 (iOS 롱프레스 메뉴 차단)
- `user-select: none` 추가 (텍스트 선택 차단)

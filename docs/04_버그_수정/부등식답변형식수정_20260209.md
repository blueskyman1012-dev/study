# 부등식 문제 답 형식 수정 (2026-02-09)

## 문제 상황
- 부등식 문제 (예: `2x - 3 > 5`)의 답이 경계값 숫자 `4`로만 출력됨
- 실제로는 `x > 4` 범위가 정답 → 4, 5, 6, 100... 모두 맞는데 `4`만 인정
- 4지선다에서 `4, 3, 7, -5` 같은 숫자가 나오면 여러 개가 맞을 수 있음

## 수정 내용

### 1. ProblemGeneratorService.js - 문제 생성 프롬프트
- `buildGeneratePrompt()`: 부등식(inequality) 주제일 때 별도 프롬프트 적용
  - 정답: `"x > 4"` (부등식 표현)
  - 선택지: `["x > 4", "x < 4", "x ≥ 4", "x ≤ 4"]` (부등호 방향/등호 변형)
  - "경계값 숫자만 정답으로 쓰지 마세요" 명시
- `buildSimilarPrompt()`: 원본 답에 부등호(`<>≤≥`)가 포함되면 부등식 규칙 적용

### 2. GeminiService.js - AI 프롬프트
- `generateSimilarProblems()`: 문제/답에 부등식 패턴 감지 시 부등식 규칙 추가
- `generateNewProblems()`: 주제에 "부등식" 포함 시 부등식 규칙 추가
- `analyzeImage()`: 이미지 분석 시에도 부등식 답 형식 규칙 추가

### 3. MonsterManager.js - 오답 자동 생성
- `generateWrongAnswers()`: 답이 부등식 표현이면 `_generateInequalityWrongAnswers()` 호출
- `_generateInequalityWrongAnswers()`: 부등호 방향/등호 유무를 변형하여 오답 3개 생성
  - 예: 정답 `x > 4` → 오답 `x < 4`, `x ≥ 4`, `x ≤ 4`

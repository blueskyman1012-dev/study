# 통계 유형/난이도별 추적 및 UI 수정

## 문제점
1. 과목별 정답/오답만 추적되고 유형(topic)별 추적이 없었음
2. 어떤 난이도에서 틀렸는지 추적이 없었음
3. ResultScreen 카드 높이 고정(380px)으로 내용이 삐져나옴
4. StatsScreen 섹션 H/I의 레이아웃이 겹침/삐져나옴

## 수정 내용

### 1. 데이터 추적 확장 (Game.js, BattleManager.js)
- `wrongByTopic` / `correctByTopic`: 유형별 오답/정답 카운트
- `wrongByDifficulty`: 난이도별 오답 카운트 (쉬움/보통/어려움)
- 정답 시 `correctByTopic[monster.topic]++`
- 오답 시 `wrongByTopic[monster.topic]++`, `wrongByDifficulty[difficulty]++`

### 2. 누적 집계 (StatsManager.js)
- `correctByTopic`, `wrongByTopic`, `wrongByDifficulty` 전체 누적 합산

### 3. 결과 화면 UI 수정 (ResultScreen.js)
- 카드 높이를 내용에 따라 동적 계산 (고정값 제거)
- 아이템 사용을 5열 1줄로 압축
- 과목별 정답/오답 + 정답률% 표시
- 유형별 오답 분석 (오답 많은 순 최대 5개)
- 난이도별 오답 분석
- 버튼 위치 카드 바로 아래로 동적 배치

### 4. 통계 화면 UI 수정 (StatsScreen.js)
- 섹션 H 아이템 사용: 2줄x3열로 정리, 높이 140→120 축소
- 섹션 I 과목별 오답: 정답/오답 바 + 수치 + 오답률을 한 줄에 배치
- 섹션 J(신규) 유형별 오답: 오답 많은 순 최대 8개, topic 이름 표시
- 섹션 K(신규) 난이도별 오답: 바 차트로 쉬움/보통/어려움별 오답 수
- 스크롤 높이 완전 동적 계산

### 5. 번역 키 추가 (i18n.js)
- `stats_topicAnalysis`: 유형별 오답 분석 (한/영/일/중)
- `stats_diffWrong`: 난이도별 오답 (한/영/일/중)

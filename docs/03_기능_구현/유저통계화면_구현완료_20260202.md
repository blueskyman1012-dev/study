# 유저 통계 화면 구현 완료

## 개요
유저별 문제 풀이 통계, 과목별 분석, 난이도 분석, 플레이 기록을 보여주는 통계 화면을 추가했다.

## 변경 내역

### 신규 파일
1. **`app/src/game/StatsManager.js`** - IndexedDB `runs` 전체 조회 후 통계 집계 매니저
   - `aggregateStats()`: totalRuns, totalClears, totalFails, totalAnswers, totalCorrect, bestCombo, totalGoldEarned, 과목별/난이도별 분포 집계
   - 결과를 `game.cachedStats`에 캐싱하여 render()에서 동기적으로 사용

2. **`app/src/game/screens/StatsScreen.js`** - 통계 화면 렌더링 (4개 카드 레이아웃)
   - 전체 통계: 총 문제 수, 정답률(%), 총 런 횟수
   - 과목별 통계: 수학/영어/국어/과학 문제 수 + 바 차트
   - 난이도별 분석: 쉬움/보통/어려움 분포 + 바 차트
   - 플레이 기록: 클리어/실패, 최고 콤보, 총 획득 골드

### 수정 파일
3. **`app/src/utils/constants.js`** - `SCREENS` 열거형에 `STATS: 'stats'` 추가
4. **`app/src/game/Game.js`**
   - StatsManager import 및 constructor에서 초기화
   - `changeScreen()`에서 STATS 진입 시 `aggregateStats()` 호출
   - `render()` switch문에 `case SCREENS.STATS` 추가
5. **`app/src/game/screens/MainScreen.js`**
   - 기존 2버튼(상점/설정) → 3버튼(통계/상점/설정) 레이아웃 변경
   - 각 버튼 폭 175px → 116px로 조정

## 설계 결정
- **async 이슈 해결**: render()는 동기 함수이므로, changeScreen() 시점에 미리 aggregateStats()를 호출하여 cachedStats에 저장
- **데이터 소스**: 기존 runs/monsters 데이터만 활용, 새 DB 필드 불필요
- **스크롤 불필요**: 4개 카드를 700px 캔버스 내에 배치 완료
